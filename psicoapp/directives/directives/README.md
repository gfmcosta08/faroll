# Directives (Camada 1 – O que fazer)

SOPs em Markdown que definem:

- **Objetivos** e resultado esperado
- **Entradas** necessárias
- **Ferramentas/scripts** a usar (em `execution/`)
- **Saídas** e formato
- **Edge cases** e exceções

Instruções em linguagem natural, como para um colaborador de nível intermediário.  
As diretivas são documentos vivos: atualize-as quando descobrir limitações de API, melhores abordagens ou erros comuns.

**Não crie novas diretivas sem permissão e não sobrescreva diretivas existentes sem o usuário pedir.**
